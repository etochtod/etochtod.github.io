__slots__

Коли об'єкт має атрибут __dict__, то це зручний та гнучкий простір імен. Можна на ходу додавати нові атрибути, як завгодно маніпулювати ними. Недоліком є повільність та займання великого обсягу пам'яті.

Якщо програміст хоче написати ефективний по пам'яті та швидкості виконання клас, то він має переробити його, додавши атрибут __slots__. Якщо ми додамо в клас такий атрибут, то він є антагоністом для __dict__. Останній не буде використовуватися. Натомість, атрибути підуть у __slots__.

class Users:
    __slots__ = ["name", "email"]

    def __init__(self, name, mail):
        self.name = name
        self.email = mail

a = Users("Alex", "alex@company.com")
print(a.__slots__) # ['name', 'email']
# __dict__ буде відсутній серед атрибутів

Ми заздалегідь прописуємо всі атрибути-дані, втрачаємо гнучкість, натомість отримуємо ефективність. Всі нащадки будуть також мати __slots__.
