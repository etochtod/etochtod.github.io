Класи та інстанси

Об'єкти поділяються на класи та інстанси. Клас потрібен для створення інстансів. Будь-який клас також грає роль інстанса, оскільки його хтось створив. Клас, який створює інші класи, називають метакласом.

Для об'єкта s

s = "Hello world!"
ланцюг хто кого створює буде наступним: 
s  ←  str  ←  type
  • s - інстанс класа str
  • str - інстанс класа type
  • type є метакласом для str 
s (instance) ← (class) str (instance) ← (metaclass) type

Творець інстанса вказаний в його атрибуті __class__.


Прикладні класи

Програміст може створити свій клас за допомогою синтаксиса:

# створюємо клас. Змінна Greeting буде вказувати на нього.
class Greeting:
    # задаємо атрибут-дані класа
    s = "Hello world!"

    # задаємо атрибут-метод класа
    def print_to_screen(self):
        print(self.s)

# створюємо інстанс класа
a = Greeting()
# виконуємо метод інстанса
a.print_to_screen() # Hello world!

Всі змінні всередині класа будуть перероблені на атрибути класа і поміщені в __dict__.

Про self буде розказано пізніше. Поки що важливо з'ясувати як виглядає створення класа та створення інстанса.


Перекриття атрибутів

Інстанси наслідують від своїх класів/метакласів атрибути.

Наприклад, інстанс a унаслідував від класа Greeting атрибут-метод print_to_screen. Інстанс a може створити собі атрибут з таким же ім'ям print_to_screen, і тоді цей атрибут перекриє атрибут класа.

a.print_to_screen = "new text"
# print_to_screen підмінений, і не буде виводити "Hello world!"
print(a.print_to_screen) # new text

Атрибут класа буде лежати під атрибутом інстанса перекритий та невидимий. Атрибут інстанса нашарувався на нього зверху. Щоб атрибут класа знов побачити, треба видалити атрибут інстанса.

del a.print_to_screen

Тоді з-під видаленого атрибута інстанса знов визирне його тьозка - атрибут класа.
