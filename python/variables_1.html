Використати синтаксис з f.

Додатково створити змінну, який буде привласнено значення, що людина одружена. Використати домовленості, обираючи назву змінної.


Змінні

Загальні відомості

  • В пітоні змінні чутливі до регістру (A відрізняється від a).
  • Декларувати не треба (перше присвоювання і є декларацією).
  • Типізація динамічна (вказувати тип змінної не треба).
a = 5 # integer
b = 5.1 # real
c = "text" # string
d = True # boolean - булеве значення (True або False).
e = [10, 20, 15] # list - список (або масив в деяких мовах).
f = (10, 20, 15) # tuple - як list, тільки його не можна міняти (але він швидкіший та компактніший).
g = {10, 20, 15} # set - множина (елементи не можуть повторюватися).
h = { # dictionary - пари "ключ":значення.
    "name": "Alex",
    "age": 27
}

На екран виводить команда print()

print(a)
# 5
print("значення b дорівнює: " + b)
# значення b дорівнює: 5.1
print(f"значення a дорівнює {a}, значення b дорівнює {b}")
# значення a дорівнює 5, значення b дорівнює 5.1

Домовленості

В 2022 році прийнято писати значення змінних пітона в стилі snake case (my_variable). Раніше був популярним camel case (myVariable), коли друге слово з великої літери, і виходить посередині горб як у верблюда. Зараз це написання в пітоні використовують для назв класів та виключень. Все інше - snake case. Він легше читається.

Заохочується використання так званих самодокументованих змінних. Наприклад, якщо нам потрібна змінна для зберігання статі людини, то небажано використовувати булеву змінну sex або gender. Краще використовувати змінну is_male. Справа в тому, що коли в коді зустрінеться таке написання

if is_male:
    щось зробити
то воно схоже на розмовну англійську (is/has), і читач на ходу зрозуміє, що is_male дорівнює True коли стать чоловіча. Змінні sex та gender залишають неоднозначність. Такі назви незручні для мов з динамічною типізацією.

За таким же принципом змінюють price на dollar_price, або id на book_id, або time на time_seconds. Це замість написання коментарів.

За потреби вводять додаткові проміжні змінні для поліпшення читабельності кода. Маневр допомагає зрозуміти код миттєво замість його тривалого аналізу.

if якась складна перевірка умови:
    щось зробити

is_mail_valid = якась складна перевірка умови

if is_mail_valid:
    щось зробити

Таким чином, за допомогою правильного називання змінних, код робиться більш читабельним та самодокументованим. На продумування назв в пітоні прийнято витрачати час.
